SRE first-contact demo

Bring an app, generate agents, inject chaos, prove remediation, score the run.

This is the handover litmus test: a new SRE arrives with an application and can see how Kagent triage v2 uses BYO agents, Grafana, GitLab, knowledge, memory, HITL, chaos, and lifecycle eval in one governed loop.

01

Intake

App profile and SRE intent become failure modes and tool needs.

02

Build

Read-only triage and bounded remediation agents are generated.

03

Prove

A lower-env pod-delete failure drives triage, HITL, recovery, and eval.

04

Learn

Memory is curator-only and KB changes are proposed through MR.

End-to-end journey

What the first-time SRE should experience

SRE asks"I want to onboard checkout-api and understand what could break."
Front doorSRE first-contact agent reads app profile, Grafana signals, GitLab path, KB runbooks, and memory posture.
Failure modesCrashLoop, dependency timeout, and pod-loss recovery are proposed.
BYO agentsRead-only triage and bounded remediation agents get explicit ToolGrants.
Chaos gateSRE approves a lower-env pod-delete test with rollback and abort conditions.
Triage + remediateSpecialists fan out, Grafana/GitLab/KB/memory evidence is gathered, writes wait for HITL.
Eval + learnRecovery is verified, lifecycle eval scores the run, weak evidence routes to review-manager.
