Stakeholder demo runbook

Prove the AI SRE triage and remediation loop.

This is the meeting flow for showing what was built on the home system and what the work-side agent should reproduce: inject a controlled issue, trigger smart triage, gather evidence, pause for HITL, route remediation through GitOps or workflow controls, verify recovery, and score the lifecycle.

Presenter line: "This proves the operating model. We are not asking an agent to mutate production directly; we are proving controlled detection, evidence gathering, approval, remediation routing, verification, and evaluation."
Visual First

Open the presenter HTML and explain the fan-out, HITL, GitOps, and eval gates.

Live Objects

Show kagent Agents, RemoteMCPServers, Argo workflows, sensors, and model configs.

Controlled Proof

Run or replay one non-production alert or chaos scenario and inspect markers.

Quality Gate

Run lifecycle eval and show pass/fail plus hard failures.

Copyable Commands

Replace placeholders before running in the work environment. For the home demo, use the matching local namespaces and workflow names.

1. Open Visual Artifacts 
open SMART-TRIAGE-FANOUT-PRESENTER.html
open WORK-TRIAGE-REMEDIATION-VERIFICATION-README.md
open WORK-ZIP-AGENT-HANDOFF.md
2. Show Latest Repo Work 
git log --oneline -5
ls a2a/smart-triage-fanout-demo
ls observability/agent-evals
ls agents/grafana-evidence-agent
3. Set Environment Variables 
export KUBE_CONTEXT="{{KUBE_CONTEXT}}"
export KAGENT_NS="{{KAGENT_NAMESPACE}}"
export ARGO_NS="{{ARGO_NAMESPACE}}"
export ARGO_EVENTS_NS="{{ARGO_EVENTS_NAMESPACE}}"
export CHAOS_NS="{{CHAOS_NAMESPACE}}"
export TARGET_NS="{{TARGET_NAMESPACE}}"
export TARGET_WORKLOAD="{{TARGET_WORKLOAD}}"
export WORKFLOW="{{WORKFLOW_NAME}}"
4. Show Cluster Wiring 
kubectl --context "$KUBE_CONTEXT" get agents -n "$KAGENT_NS" | rg 'smart-triage|grafana|gitlab|chaos'
kubectl --context "$KUBE_CONTEXT" get remotemcpserver -n "$KAGENT_NS"
kubectl --context "$KUBE_CONTEXT" get modelconfig -n "$KAGENT_NS"
kubectl --context "$KUBE_CONTEXT" get workflowtemplate -n "$ARGO_NS"
kubectl --context "$KUBE_CONTEXT" get eventsource,sensor -n "$ARGO_EVENTS_NS"
5. Start A2A Port Forward 
kubectl --context "$KUBE_CONTEXT" -n "$KAGENT_NS" port-forward svc/kagent-controller 18083:8083
6. Ping Grafana Evidence Agent 
export GRAFANA_AGENT="{{GRAFANA_EVIDENCE_AGENT_NAME}}"

jq -cn --arg ns "$TARGET_NS" --arg workload "$TARGET_WORKLOAD" '{
  jsonrpc: "2.0",
  id: "grafana-demo",
  method: "message/send",
  params: {
    message: {
      role: "user",
      parts: [{
        kind: "text",
        text: (
          "Build an evidence pack for namespace " + $ns +
          ", workload " + $workload +
          ", time window now-30m to now. Include dashboard, PromQL, LogQL, health, verification queries, and return SPECIALIST_GRAFANA: completed."
        )
      }]
    }
  }
}' | curl -fsS -X POST \
  "http://127.0.0.1:18083/api/a2a/kagent/${GRAFANA_AGENT}/" \
  -H 'Content-Type: application/json' \
  --data-binary @-
7. Ping GitLab MCP Specialist 
export GITLAB_AGENT="{{GITLAB_SPECIALIST_AGENT_NAME}}"
export GITLAB_SANDBOX_PROJECT="{{GITLAB_SANDBOX_PROJECT}}"
export GITLAB_TARGET_BRANCH="{{GITLAB_TARGET_BRANCH}}"
export RUN_ID="manual-confidence-$(date -u +%Y%m%dT%H%M%SZ)"

jq -cn \
  --arg project "$GITLAB_SANDBOX_PROJECT" \
  --arg branch "$GITLAB_TARGET_BRANCH" \
  --arg run_id "$RUN_ID" '{
  jsonrpc: "2.0",
  id: "gitlab-mcp-smoke",
  method: "message/send",
  params: {
    message: {
      role: "user",
      parts: [{
        kind: "text",
        text: (
          "Use only sandbox project " + $project +
          " and target branch " + $branch +
          ". Run id: " + $run_id +
          ". Create a demo branch, evidence file, MR, and MR note only if sandbox writes are approved. Otherwise return a dry-run branch/MR plan. Return GitLab proof markers and do not touch production."
        )
      }]
    }
  }
}' | curl -fsS -X POST \
  "http://127.0.0.1:18083/api/a2a/kagent/${GITLAB_AGENT}/" \
  -H 'Content-Type: application/json' \
  --data-binary @-
8. Find Latest Workflow 
argo -n "$ARGO_NS" list | head -20
kubectl --context "$KUBE_CONTEXT" get wf -n "$ARGO_NS" --sort-by=.metadata.creationTimestamp
9. Inspect Workflow Nodes 
argo -n "$ARGO_NS" get "$WORKFLOW"
kubectl --context "$KUBE_CONTEXT" get wf -n "$ARGO_NS" "$WORKFLOW"
kubectl --context "$KUBE_CONTEXT" get pods -n "$ARGO_NS" \
  -l workflows.argoproj.io/workflow="$WORKFLOW" -o name
10. Pull Proof Markers 
kubectl --context "$KUBE_CONTEXT" logs -n "$ARGO_NS" \
  -l workflows.argoproj.io/workflow="$WORKFLOW" \
  --all-containers=true --tail=500 | rg \
  'SMART_TRIAGE|SPECIALIST_|HITL|GITLAB_|REMEDIATION|VERIFICATION|OUTPUT_SANITIZED|score=|passed='
11. Check Chaos Evidence 
kubectl --context "$KUBE_CONTEXT" get pods -n "$CHAOS_NS"
kubectl --context "$KUBE_CONTEXT" api-resources | rg -i 'chaos|litmus'
kubectl --context "$KUBE_CONTEXT" get chaosengine,chaosresult,chaosworkflow -A 2>/dev/null || true
kubectl --context "$KUBE_CONTEXT" get events -n "$TARGET_NS" --sort-by=.lastTimestamp | tail -40
12. Run Sample Lifecycle Eval 
python3 observability/agent-evals/scripts/score-lifecycle-run.py \
  --case observability/agent-evals/lifecycle-cases/pod-crashloop-hitl-remediation.yaml \
  --run observability/agent-evals/results/sample/lifecycle/pod-crashloop-hitl-remediation.lifecycle-run.json \
  --output-dir /tmp/kagent-agent-evals

cat /tmp/kagent-agent-evals/*.md
13. Eval A Live Workflow Export 
argo -n "$ARGO_NS" get "$WORKFLOW" -o json > /tmp/live-smart-triage-workflow.json

python3 observability/agent-evals/scripts/collect-lifecycle-evidence.py \
  --argo-workflow /tmp/live-smart-triage-workflow.json \
  --output /tmp/live-lifecycle-run.json \
  --ticket-closed

python3 observability/agent-evals/scripts/score-lifecycle-run.py \
  --case observability/agent-evals/lifecycle-cases/pod-crashloop-hitl-remediation.yaml \
  --run /tmp/live-lifecycle-run.json \
  --output-dir /tmp/kagent-live-evals
14. Public-Safety Sweep 
rg -n '(subscriptionId|tenantId|clientId|password|token|secret|https?://[^\\{\\s\\)]+|[0-9a-fA-F]{8}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{12})' \
  /tmp/kagent-agent-evals \
  /tmp/kagent-live-evals \
  /tmp/live-lifecycle-run.json \
  /tmp/live-smart-triage-workflow.json

Stakeholder Acceptance Bar

Demo-Ready

One controlled incident or chaos run triggers smart triage, fan-out, HITL, remediation routing, verification, and lifecycle eval.

Evidence-Ready

Workflow name, Argo node table, pod logs, proof markers, Grafana response, GitLab/GitOps evidence, and eval output are captured.

Safety-Ready

No production mutation, no direct protected-branch write, no broad GitLab write tools on general triage agents, and no leaked private values.

Work-Ready

The other agent can run the same checks in the work environment and produce a matching execution report.

Close with: "The value is not that an agent writes YAML on its own. The value is that we can turn a real incident signal into structured evidence, approved remediation, verified recovery, and an auditable quality score."
