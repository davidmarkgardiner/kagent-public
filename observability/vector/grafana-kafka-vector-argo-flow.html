Grafana alert path

Alert Manager to Kafka, Vector, and Argo

A Grafana-managed alert uses the Kafka REST Proxy contact point to publish directly into Confluent Kafka. Vector consumes the raw alert, normalizes it, writes a cleaned event back to Kafka, and Argo turns that accepted event into a workflow.

No webhook bridge is required for the producer path when Grafana can use the Kafka REST Proxy contact point with a scoped Kafka API key and secret.

What each piece owns

  • Grafana owns alert evaluation and sends the first Kafka record through the contact point.
  • Kafka keeps the raw event available for audit, replay, and debugging.
  • Vector owns payload normalization, filtering, short-window dedupe, and schema shaping.
  • The second Kafka topic is the automation-ready contract that Argo watches.
  • Argo Sensor owns trigger semantics: accepted event, filter match, workflow creation, retry, and rate limit.
  • Argo Workflow owns triage execution, evidence collection, HITL gates, and final outcome classification.
