## Fix required to get here (important for the work bundle)

kagent **0.9.9** pins the Go ADK agent-runtime image to the `cr.kagent.dev` mirror, which does
**not** serve that repository:

```
err: while creating "kagent" OCI bundle: in remote.Image:
GET https://cr.kagent.dev/v2/kagent-dev/kagent/golang-adk/manifests/sha256:e01479...
NAME_UNKNOWN: repository name not known to registry
```

The **identical digest is available on GHCR** (`ghcr.io/kagent-dev/kagent/golang-adk`,
verified HTTP 200). Two things to note:

1. This is a *different* image from `substrateWorkerPool.ateomImage`
   (`ghcr.io/kagent-dev/substrate/ateom-gvisor:v0.0.6`, the worker host) — that one is a helm
   value and pulled fine. The broken one is the **agent runtime inside the actor**, which 0.9.9
   does **not** expose as a helm value.
2. Workaround used here — repoint the ActorTemplate to GHCR (same digest):
   ```bash
   # stop kagent-controller reverting the field while ate-controller boots the golden actor
   kubectl -n kagent scale deploy/kagent-controller --replicas=0
   kubectl -n kagent patch actortemplate hello-substrate --type=json \
     -p='[{"op":"replace","path":"/spec/containers/0/image",
           "value":"ghcr.io/kagent-dev/kagent/golang-adk@sha256:e01479b52280b0eae9e2808cc68392ba98fd737782496ff256847257e6bb8ed1"}]'
   # golden snapshot captures; then scale controller back
   kubectl -n kagent scale deploy/kagent-controller --replicas=1
   ```
   Once the golden snapshot exists, `SandboxAgent` stays `Ready` even though the controller
   re-writes the image field back to `cr.kagent.dev`.

**For a real deployment (incl. AKS):** mirror `ghcr.io/kagent-dev/kagent/golang-adk` into your
own registry (ACR) and, better, raise/track the upstream registry-path bug so the pinned image
resolves without the patch. A newer kagent release may already point at a valid path — check
before pinning to 0.9.9.

## gVisor verdict

gVisor checkpoint/restore **worked on plain kind** (containerd-in-Docker, kernel 6.8) with no
special node runtime — substrate ships its own runsc (fetched from `gs://gvisor/...`) and the
ateom-gvisor userspace. This is the encouraging signal for AKS: the mechanism itself is
portable; the AKS gate is admission policy (privileged pods / Kyverno), not gVisor availability.
See [`../WORK-CLUSTER-ADAPTATION.md`](../WORK-CLUSTER-ADAPTATION.md).
