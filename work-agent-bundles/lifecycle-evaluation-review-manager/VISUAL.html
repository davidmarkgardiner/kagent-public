Work-agent bundle

Lifecycle Evaluation Review Manager

This bundle gives another agent the design, build plan, and evidence checklist for implementing offline and online lifecycle evaluation in a work environment.

Meeting actions covered

Evaluation framework

Cases, scorers, reports, metrics, dashboards, alerts, and Argo runtime are mapped.

Offline and online eval

Offline is CI/replay. Online is Argo post-run audit and gating.

Key metrics

metrics.py owns metric names, labels, escaping, and Prometheus output.

Architecture

Separate reusable evaluator step, inline public-image runtime, ConfigMap-mounted scripts.

Storage and access

Raw evidence, lifecycle JSON, eval JSON, Markdown, metrics, and dashboards are separated.

Retention and traceability

Run IDs, workflow names, scores, hard failures, and ticket links form the audit trail.

Build and verify flow

1Read bundle

Start with front sheet, meeting coverage, architecture, storage/access, and request YAML.

2Render runtime

kubectl kustomize observability/agent-evals

3Score pass

Run lifecycle scorer against a known good sample.

4Score fail

Run below-threshold or hard-failure sample and capture non-success evidence.

5Export metrics

Generate Prometheus text through the independent metrics library.

6Route review

Keep ticket open or create review-manager route when eval fails.
