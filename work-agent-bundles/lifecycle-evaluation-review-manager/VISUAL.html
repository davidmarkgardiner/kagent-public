Work-agent bundle

Lifecycle Evaluation Review Manager

This bundle gives another agent the design, build plan, and evidence checklist for proving chaos-event to triage to lifecycle evaluation to GitLab documentation in a work environment.

Meeting actions covered

Evaluation framework

Cases, scorers, reports, metrics, dashboards, alerts, and Argo runtime are mapped.

Offline and online eval

Offline is CI/replay. Online is Argo post-run audit and gating.

Key metrics

metrics.py owns metric names, labels, escaping, and Prometheus output.

Architecture

Separate reusable evaluator step, inline public-image runtime, ConfigMap-mounted scripts.

Storage and access

Raw evidence, lifecycle JSON, eval JSON, Markdown, metrics, and dashboards are separated.

Retention and traceability

Run IDs, workflow names, scores, hard failures, and ticket links form the audit trail.

Chaos event proof

Argo Events, Litmus, Kubernetes events, or workflow watch can trigger the phase-1 proof path.

GitLab evidence

The resulting issue, MR, comment, or report captures event, triage, evaluation, and next action.

Grafana optional

Grafana alert triggering is useful later, but not required for the first end-to-end proof.

Build and verify flow

1Resolve variables

Read ../SHARED-VARIABLES.md and confirm namespaces, GitLab targets, MCP names, and run IDs.

2Observe chaos

Use Argo Events, Litmus, Kubernetes events, or workflow watch to capture the chaos action.

3Start triage

Send the observed payload to the Kagent triage path and capture run or workflow IDs.

4Score lifecycle

Evaluate the result with the chaos-pod-delete lifecycle case and hard gates.

5Route review

Keep weak runs open and create a review-manager route with owner and next action.

6Update GitLab

Attach event, triage, score, evidence links, and next action to an issue, MR, comment, or report.
