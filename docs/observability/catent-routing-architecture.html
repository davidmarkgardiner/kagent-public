Catent alert routing

From alert noise to the right first agent.

Alertmanager sends the webhook into Vector. Vector normalizes and labels the event. Argo Sensors do coarse lane filtering. The Workflow receives the full envelope and performs the final delegation to the best specialist.

Operating decision

Keep one normalized actionable topic first. Use fields like queue_priority, route_domain, namespace_class, and target_agent to route before splitting Kafka topics.

Sensors stay simple

Sensors filter by schema, firing state, priority, and coarse lane. They should not parse every event message or own the full routing policy.

Workflow owns delegation

Workflow receives the whole normalized envelope and decides the first specialist using namespace, event type, reason, message, and Vector hints.

Split topics later

Only split Kafka topics when priority traffic is delayed, ownership/ACLs differ, or retention/compliance needs a hard boundary.
