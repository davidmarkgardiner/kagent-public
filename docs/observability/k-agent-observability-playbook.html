MIL-124 verification

K-Agent observability from Alloy to Grafana

Use this page to prove logs, metrics, token usage, dashboards, and alerts for kagent and agentgateway in a work environment. The checks map directly to the repo manifests and Grafana dashboard.

What To Prove

Logs

kagent and gateway logs land in Loki

Verify with LogQL for {namespace="kagent"} and {namespace="kgateway-system"}.

Metrics

Gateway and container metrics land in Mimir/Prometheus

Verify Envoy request metrics, kagent pod restarts, CPU, and memory queries.

Tokens

Token usage is visible or fallback is explicit

Prefer agentgateway_gen_ai_client_token_usage_*. If absent, use structured kagent log token fields until gateway emits counters.

Dashboard Visual Checks

agentgateway Tokens Per Minute

Should move after an LLM-backed request when the gateway emits token counters. Blank means no traffic, missing token metrics, or remote_write failure; use the log fallback if token fields are present there.

agentgateway Upstream Response Rate

Confirms gateway metrics are landing. Look for response class split and upstream names.

Running kagent Pods

Confirms kube-state-metrics and container metrics are available for the kagent namespace.

kagent Restarts Last Hour

Useful early warning for unstable agents or controllers.

kagent API and A2A Logs

Confirms Loki labels and JSON parsing are usable from Grafana.

agentgateway Suspicious Logs

Surfaces reset, timeout, model, token, and upstream patterns for investigation.

Copy-Paste Queries

Token usage by model

sum by (gen_ai_request_model, gen_ai_token_type) (
  rate(agentgateway_gen_ai_client_token_usage_sum[5m]) * 60
)

Gateway response rate

sum by (envoy_cluster_name, envoy_response_code_class) (
  rate(envoy_cluster_external_upstream_rq_xx[5m])
)

kagent restarts

sum by (namespace, pod, container) (
  increase(kube_pod_container_status_restarts_total{namespace="kagent"}[1h])
)

kagent logs

{namespace="kagent"} | json | line_format "{{.level}} {{.logger}} {{.path}} status={{.status}}"
