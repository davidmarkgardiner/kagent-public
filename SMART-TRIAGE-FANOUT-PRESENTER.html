TL;DR

The demo takes an Alertmanager replay, fans out to eight specialists, synthesizes the evidence, suspends for human review, resumes, and emits proof markers.

It is ready as a first PoC PR, with a clear caveat: the public demo uses synthetic specialist evidence where live public-safe backends are not present. The work implementation should wire the same contracts to real read-only Kubernetes, network/Hubble, Grafana, GitLab/GitOps, KB, deployment, policy, and trace paths.

The GitLab write path has also been proven separately from inside kagent: a mounted MCP shim created a sandbox branch, two commits, an MR, and an MR note in GitLab.

Presenter line: This proves the orchestration and safety pattern, plus a sandbox GitOps write path. It does not claim production remediation.

Lift-And-Shift Rules

Parameterize first

Replace context, namespaces, model config, MCP server names, GitOps sandbox, HITL front door, and retention.

Keep specialists read-only

Kubernetes, network, and Grafana agents should gather evidence only. No apply, delete, patch, exec, or restart tools.

GitOps stays gated

Start with sandbox issue/MR/diff creation. Move to real branch/MR only after HITL and review are proven.

Prove runtime, not status

Accepted=True is not enough. Run real A2A, model, Grafana, network, GitOps, and Argo suspend/resume checks.

Capture evidence

No success claim without workflow name, node table, pod logs, proof markers, HITL record, and failure class if any.

Keep it public-safe

No secrets, tokens, private hostnames, internal URLs, subscription IDs, tenant IDs, or real credentials in docs.

Current limitation: this public PoC uses synthetic specialist evidence. The GitLab write path is proven separately through a sandbox kagent-mounted MCP shim. The work implementation should keep the same proof contract while replacing synthetic sources and shim auth with approved live integrations.
