## 1. What this project is

An event-driven Kubernetes triage system. Cluster events (or Prometheus alerts) flow into Argo Events, get fanned out by namespace, dispatched to a kagent **specialist agent** over A2A (JSON-RPC), and the agent's diagnosis is filed as a GitLab issue. Optional HITL gate before any write action. Notifications are environment-specific (Telegram at home, Teams at work — out of scope here).

Two deployment shapes:
- **Worker cluster** — kagent + agentgateway local, native k8s tools, triages app namespaces.
- **Management cluster** — system-only specialists (flux-system, cert-manager, istio, gatekeeper, kube-system), may use AKS-MCP for cross-cluster reads.

## 3. End-to-end flow (the bit you asked about)

```
[Source]                [Transport]              [Routing]            [Dispatch]                  [Output]
Alloy (k8s events) ─┐                          ┌─ Argo Events ──┐
                    ├─► Event Hub (OTLP/Kafka)─┤  Sensor        ├──► Argo Workflow (DAG) ──► kagent agent (A2A) ──► GitLab issue
Prom AlertManager ──┴─► Webhook ───────────────┘  rate-limited 5/min  parse → classify → dispatch    │              ├─► Telegram (home)
                                                                                                     │              └─► Teams (work, out of scope)
                                                                              ▲                      │
                                                                              │                      ▼
                                                                       (HITL suspend) ◄─── card ──► mock Teams bot ──► Argo Events callback ──► resume/stop
```

### Stage-by-stage with file paths

1. **Collect (Alloy on the workload cluster)**
   - `aks-mgmt-stack/k8s-event-triage/working-config/03-alloy.yaml` — production Alloy v1.12.2 (ConfigMap + Deployment + Service + RBAC). Uses `loki.source.kubernetes_events`, exports OTLP to Event Hub. Connection string is a separate Secret.
   - Local equivalent: `aks-mgmt-stack/k8s-event-triage/workload-cluster/02-alloy-config.yaml` + `03-alloy-deployment.yaml`.

2. **Transport** — Azure Event Hub topic `k8s-events` (Kafka 2.1.0 over SASL_PLAIN). Three consumer groups: critical / warnings / infra.

3. **Consume — Argo Events Kafka EventSource**
   - `aks-mgmt-stack/k8s-event-triage/eventhub-otlp-pipeline/02-eventsource.yaml`

4. **Route — Sensor (rate-limited 5/min)**
   - `…/eventhub-otlp-pipeline/03-sensor.yaml`

5. **Parse + classify + dispatch — Argo WorkflowTemplate**
   - `…/eventhub-otlp-pipeline/04-workflow-template.yaml` — `parse-otlp` (jq filter) → `classify-and-alert` fan-out (`withParam`) → A2A call.
   - OTLP shape reference: `…/eventhub-otlp-pipeline/OTLP-PAYLOAD-REFERENCE.md`. Routing-logic test: `tier-critical/test-routing-logic.sh` (5 OTLP fixtures, 8 assertions).

6. **Triage — kagent agent over A2A**
   - Call shape (proven, do **not** use the Sessions API — broken):
     `POST /api/a2a/kagent/{agent}/` body `{jsonrpc:"2.0",method:"message/send",params:{message:{role:"user",parts:[{kind:"text",text:"…"}]}}}`
   - Agents (per namespace) live at:
     - Worker: `kagent-triage/worker-cluster-bundle/agent-*.yaml` (11 of them)
     - Management: `kagent-triage/aks/{kube-system,istio-system,istio-ingress,flux-system,gatekeeper-system}-agent.yaml`
     - Demo / dev: `kagent-triage/01-test-agent.yaml`, `cert-manager-agent.yaml`, `kyverno-agent.yaml`, `external-secrets-agent.yaml`, `kro-agent.yaml`, `reloader-agent.yaml`
   - ModelConfig pointing at remote agentgateway: `kagent-triage/aks/modelconfig-remote-litellm.yaml`.

7. **GitLab ticket creation**
   - Workflow step `create-gitlab-issue` (node.js image) inside `aks-mgmt-stack/holmes-argoworkflows/local-llm-analysis-only.yaml` and the canonical embedded step `aks-mgmt-stack/k8s-event-triage/working-config/10-gitea-issue-step.yaml`.
   - Reads `/work/llm-response.json` (shared `volumeClaimTemplate: work, 64Mi`), POSTs to `/api/v4/projects/{PROJECT_ID}/issues` with header `PRIVATE-TOKEN: $GITLAB_TOKEN`, writes URL/IID to `/work/gitlab-issue-url.txt`.
   - Token bootstrap Job: `…/working-config/11-gitea-credentials-secret.yaml`.

8. **Three Argo workflow templates that own the dispatch shape** (`aks-mgmt-stack/holmes-argoworkflows/`)
   - `local-llm-analysis-only.yaml` — analysis only, ~67 s, no cluster access.
   - `holmes-remediation.yaml` — kagent + AKS-MCP `call_kubectl`, can patch.
   - `kagent-sre-workflow.yaml` — A2A to `sre-triage-agent` (read-only) or `sre-remediation-agent` (write, HITL-gated).

9. **HITL gate (optional, before write actions)** — `ai-platform/teams-hitl/`
   - Workflow suspends → posts Adaptive Card to (mock) Teams bot → user clicks → callback hits `teams-hitl-callback-eventsource-svc.argo-events:12000/approval-callback` → Argo Events sensor resumes / stops.
   - Hardened: HMAC verification, Istio VirtualService + AuthorizationPolicy.
   - Smoke test: `ai-platform/teams-hitl/smoke-test.sh [--approve|--reject|--expire]` — 60 s end-to-end.

## 7. Common gotchas (from `working-config/GOTCHAS.md` + memory)

- **A2A**: trailing slash required, `kind: "text"` required on parts, Sessions API broken — use `message/send`.
- **AKS-MCP RemoteMCPServer**: must specify `toolNames` list — `None` causes ValidationError.
- **kagent**: model prefix `openai/qwen3-14b`, env `OPENAI_API_BASE` (not `OPENAI_BASE_URL`), API requires `context` field. Don't `kubectl edit` via MCP — use `patch`/`set`. Use `jq -n --arg` for JSON in shell.
- **EventBus**: native NATS, not JetStream. JetStream caused issues per GOTCHAS #13.
- **inotify**: {{CLUSTER_NAME}} defaults are too low (128); raised to 512 via sysctl (not persistent).
- **Storage**: longhorn unreliable for short-lived workflow PVCs — use `local-path`.
- **Models**: Qwen 14B fails multi-step orchestration (~0/6 tool calls). Kimi works (6/6). Use red cluster + Kimi for coordinator-style PoCs; {{CLUSTER_NAME}} + Qwen 14B for single-step triage only.
- **Prompt safety**: anchor namespace explicitly (`CRITICAL: use exact namespace "X"`) — Qwen 14B will otherwise typo namespace names.
