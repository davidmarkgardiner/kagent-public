Architecture decision · evidence-first agent triage

Keep the evidence with the incident.

Worker clusters collect, redact and correlate logs and Kubernetes events locally. A management cluster consumes the bounded evidence stream, performs durable deduplication, calls the read-only triage agent and creates one auditable ticket.

Decision

Use Alloy → Vector → Kafka as the primary trigger path for log and Kubernetes-event triage. Keep Grafana/Loki for search and Alertmanager for human-facing metric paging.

Important nuance: this is not an argument to remove Alertmanager. It is an argument not to force log/event triage through an alert abstraction that has already discarded the explaining evidence.

Responsibility split contains alert storms

Worker: keep egress small and useful

  1. Attach cluster, namespace, workload, pod and container identity.
  2. Redact secrets/PII and cap lines/bytes before Kafka.
  3. Merge matching log and event candidates under a stable fingerprint.
  4. Drop normal lifecycle noise; buffer boundedly if Kafka is unavailable.

Management: make the only durable decision

  1. Validate schema, cluster identity, time and evidence size.
  2. Atomically claim the fingerprint for 24 hours.
  3. Enrich with inventory, deployment and approved-runbook context.
  4. Throttle agents by priority and write an idempotent ticket.

Controls that matter

Correlate

Use a stable incident fingerprint, not individual log lines. The same crash-loop log and BackOff event should become one candidate.

Deduplicate durably

Vector’s cache reduces an immediate burst. A central TTL store is the authority that prevents repeat agent spend and duplicate tickets.

Protect the agent

Use Kafka lag, priority lanes, Argo parallelism, per-cluster quotas and a circuit breaker for model/A2A failure.

Quarantine safely

Unknown schema, over-sized payloads and parse failures go to a DLQ with metrics and replay—not silent discard.

Isolate tenants

Authenticated worker identities can publish only to their scoped topics. Management validates topic-to-cluster mapping.

Measure it

Track collection-to-Kafka time, oldest-message age, suppression rate, redaction failures, agent latency and ticket retries.

What the red proof establishes—and what it does not

Proven on red

Alloy → Vector → Redpanda → Argo → read-only kagent → GitLab worked for an actual log and a Kubernetes BackOff event. A replay skipped agent and ticket steps.

Must be designed for fleet

Cross-cluster identity, ACLs/mTLS, topic ownership, external durable TTL store, retention/data classification, network partitions and SLOs.

Deliberately not included

Automatic remediation. The agent triages only; all mutation remains an explicit, policy- and human-approved workflow.
